# LeapGrowNotes — 本地知识库 Blueprint

> 本地优先、非 RAG、支持 LLM、多设备同步的个人知识库系统
>
> **版本**: v2.0.0  
> **最后更新**: 2026-05-01  
> **实现状态更新**: 2026-05-01

---

## 1. 项目愿景

构建一个 **本地优先（local-first）** 的个人知识库，原生支持 Markdown、代码文件和纯文本。通过 **非 RAG** 的方式（全量上下文注入 / 结构化摘要 / 滑动窗口）将知识提供给大语言模型，同时支持多设备间的数据同步。

## 4. 模块设计

### 4.1 存储层 ✅ 已实现

**方案：文件系统 + SQLite 元数据库**

- **文件系统**：知识内容以原始文件形式存储在本地目录
  - ✅ 保持文件原始格式，便于用其他编辑器直接访问
  - ✅ 目录结构即分类结构
- **SQLite**：存储元数据与全文索引
  - ✅ 文件路径、标题、标签、创建/修改时间
  - ✅ FTS5 全文搜索索引
  - ✅ 积分/统计/会话等业务数据

### 4.2 索引与检索（非 RAG）

**核心理念：不使用向量嵌入，而是基于结构化索引 + 智能上下文组装**

#### 4.2.1 全文搜索 ✅ 已实现

- ✅ 使用 SQLite FTS5 实现全文检索（17个搜索相关命令）
- ✅ 中文分词支持（FTS5 trigram 分词器，支持中日韩字符子串匹配）
- ✅ 支持文件名+内容搜索、搜索建议

#### 4.2.2 结构化索引 ✅ 已实现

- ✅ **标签系统**：YAML frontmatter tags 解析（支持 `tags: [a, b]` 和 `tags:\n- a` 两种格式）
- ✅ **文件树索引**：基于目录路径的层级分类
- ✅ **链接图谱**：`[[wikilink]]` 解析 + Backlinks/Outlinks
- ✅ **代码索引**：正则提取函数名/类名/导出符号（支持 Python/JS/TS/Rust/Go/Java/C/C++）

#### 4.2.3 智能上下文组装 ❌ 未实现

LLM 上下文组装引擎尚未开发（依赖 LLM 集成）。

### 4.3 LLM 集成 ❌ 未实现

多后端适配器（OpenAI / Anthropic / DeepSeek / Ollama）和上下文注入系统尚未开发。

### 4.4 多设备同步 🔶 部分实现

#### ✅ 已实现
- Git 集成（init/status/commit/history/checkpoint/restore/diff，基于 git2 crate）
- Git autocommit（设置中可开启，通过 reactor 监听文件变更触发）

#### ❌ 未实现
- Git auto push/pull（远端管理）
- 冲突检测与解决 UI
- Syncthing 集成

### 4.5 前端 / 用户界面 ✅ 大部分已实现

**技术栈：Tauri 2 (Rust + SvelteKit + Svelte 5)**

- ✅ **桌面端**：Tauri 2 打包，macOS / Windows / Linux
- ✅ **编辑器**：Milkdown (ProseMirror) + CodeMirror 6 代码块 + Prism 语法高亮
- ✅ **搜索面板**：Omnibar 全文搜索（文件/内容/命令/设置）
- ❌ **LLM 对话面板**：未实现

## 8. 关键设计决策记录

### 为什么选择非 RAG？

| RAG 方案                   | 本项目方案（非 RAG）         |
| -------------------------- | ---------------------------- |
| 需要向量数据库             | 仅需 SQLite                  |
| Embedding 计算开销大       | 轻量级全文索引               |
| 语义搜索可能召回不相关内容 | 精确关键词 + 结构化关联      |
| 黑盒召回过程               | 透明的上下文组装逻辑         |
| 适合海量文档               | 适合个人级知识库（数千文件） |

### 为什么选择 Tauri 而非 Electron？

- 打包体积小（~10MB vs ~150MB）
- 内存占用低
- Rust 后端性能优异，适合文件索引和搜索
- Tauri 2 支持移动端

### 二次开发基座

基于 [Otterly](https://github.com/ajkdrag/otterly)（MIT 许可）进行二次开发，保留其成熟的文件管理、搜索、编辑器、Git 集成等基础功能。
