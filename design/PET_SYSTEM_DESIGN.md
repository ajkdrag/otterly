# 🐾 LeapGrowNotes 电子宠物系统 — 产品设计规划文档

> **版本**: v1.0 Draft  
> **日期**: 2026-05-01  
> **参考**: [gitanimals](https://github.com/git-goods/gitanimals) — GitHub 贡献动物系统  
> **定位**: LeapGrowNotes 内置陪伴型电子宠物，与知识成长体系深度绑定

---

## 1. 产品愿景

> 让每一次学习和笔记行为，都转化为宠物的成长与陪伴。宠物是用户知识旅程的可视化伙伴，随着主人的持续学习而进化、成长、获得新能力。

### 1.1 核心理念

| 维度 | 说明 |
|------|------|
| 🎯 目标 | 通过宠物陪伴机制，增强用户粘性和学习动力 |
| 🔗 绑定 | 宠物成长与现有积分系统（Points）和成长等级（Growth Levels）深度联动 |
| 🏠 场景 | 宠物常驻于 App 界面，全程陪伴用户的笔记和学习过程 |
| 🌱 成长 | 宠物有独立的生命周期：孵化 → 成长 → 进化 → 成熟 |
| 🎨 个性 | 每只宠物有独特的外观、性格、技能，反映主人的学习风格 |

### 1.2 与 gitanimals 的差异化

| gitanimals | LeapGrowNotes 宠物系统 |
|------------|----------------------|
| 基于 GitHub 贡献热力图 | 基于笔记/学习行为数据 |
| 展示在 GitHub Profile | 常驻于 App 内界面 |
| 静态/简单动画展示 | 实时互动 + 情绪反馈 + 动画 |
| 贡献数量驱动 | 多维度行为驱动（阅读、写作、NLP分析等）|
| 单一宠物 | 多宠物收集 + 进化体系 |
| Web 展示为主 | 桌面端原生体验 |

## 2. 宠物种类设计

### 2.1 初始宠物蛋（Starter Eggs）

用户首次使用时选择一枚宠物蛋进行孵化，共 **5 种基础系列**：

| 编号 | 蛋类型 | 孵化宠物 | 属性倾向 | 对应学习风格 |
|------|--------|---------|---------|-------------|
| 🥚1 | 墨水蛋 | **墨灵 (InkSprite)** | 📝 写作型 | 偏好创作笔记、长文写作 |
| 🥚2 | 书页蛋 | **卷卷 (ScrollPup)** | 📖 阅读型 | 偏好阅读、文件浏览 |
| 🥚3 | 齿轮蛋 | **码仔 (CodeKit)** | 💻 技术型 | 偏好代码块、技术笔记 |
| 🥚4 | 星云蛋 | **思思 (ThinkCloud)** | 🧠 思考型 | 偏好链接、知识图谱 |
| 🥚5 | 种子蛋 | **芽芽 (SproutBud)** | 🌱 全能型 | 均衡发展 |

### 2.2 进化路线（Evolution Tree）

每只基础宠物有 **4 个进化阶段**，对应成长等级：

```
阶段1: 幼崽期 (Baby)      — Lv.0~5  (👶 婴儿 ~ 🎒 小学生)
阶段2: 成长期 (Teen)       — Lv.6~12 (📗 初中生 ~ 🎓 大学生)  
阶段3: 成熟期 (Adult)      — Lv.13~20(🔬 研究助理 ~ 📚 副教授)
阶段4: 传说期 (Legendary)  — Lv.21~29(🏅 教授 ~ 👑 学术泰斗)
```

#### 墨灵 (InkSprite) 进化示例：

```
墨灵幼崽 🐣 → 小墨精 ✒️ → 文墨师 📜 → 传说·翰墨龙 🐉
```

#### 分支进化

在阶段2→阶段3时，根据主人的**主要行为模式**触发分支进化：

| 主要行为 | 分支方向 | 外观变化 |
|---------|---------|---------|
| 写作为主 (>60%) | 创作系 | 笔/书卷元素增加 |
| 阅读为主 (>60%) | 学者系 | 眼镜/书本元素增加 |
| 代码为主 (>60%) | 工程系 | 齿轮/电路元素增加 |
| 链接/图谱为主 (>60%) | 智者系 | 星辰/网络元素增加 |
| 均衡发展 | 全能系 | 彩虹/光环元素增加 |

## 3. 宠物属性系统

### 3.1 核心属性

```typescript
interface Pet {
  id: string;
  name: string;                    // 用户自定义名字
  species: PetSpecies;             // 种类
  stage: EvolutionStage;           // 进化阶段 1~4
  variant: EvolutionVariant;       // 进化分支

  // 生命属性
  happiness: number;               // 快乐值 0~100
  energy: number;                  // 活力值 0~100
  hunger: number;                  // 饱腹度 0~100
  bond_level: number;              // 亲密度 0~1000

  // 成长属性
  exp: number;                     // 经验值
  level: number;                   // 宠物自身等级 1~50
  
  // 个性属性
  personality: Personality;        // 性格特征
  mood: Mood;                      // 当前心情
  skills: PetSkill[];              // 已学会的技能
  
  // 外观
  accessories: Accessory[];        // 装饰品/装备
  skin_variant: string;            // 皮肤变体
  
  // 时间
  born_at: number;                 // 出生时间
  last_fed_at: number;             // 上次喂食
  last_interaction_at: number;     // 上次互动
}
```

### 3.2 性格系统

宠物在孵化时随机获得 1~2 个性格特征，影响行为和反应：

| 性格 | 效果 | 表现 |
|------|------|------|
| 🌟 勤勉 | 主人写笔记时额外 +10% 经验 | 会主动翻书、做笔记动画 |
| 😴 慵懒 | 闲置时不扣快乐值 | 经常打瞌睡、伸懒腰 |
| 🤓 好奇 | NLP分析时额外 +15% 经验 | 会探头看屏幕、提问号动画 |
| 🎭 活泼 | 互动时快乐值恢复更快 | 跳跃频率更高、表情丰富 |
| 📚 博学 | 阅读完成时额外 +10% 经验 | 会戴眼镜、翻书动画 |
| 💪 坚韧 | 连续使用天数加成更高 | 不轻易哭泣、耐力姿态 |

### 3.3 心情系统

宠物的心情受多种因素影响，通过表情和动画实时反映：

```
😄 开心  — 主人活跃学习中，属性良好
😊 满足  — 基本需求被满足
😐 平静  — 一般状态
😟 无聊  — 长时间未互动 (>2小时)
😢 难过  — 属性值过低 / 长时间未使用 (>24小时)
😴 睡眠  — 夜间时段 / 主人长时间未操作 (>30分钟)
🤩 兴奋  — 达成成就 / 进化 / 获得新技能
🎉 庆祝  — 主人升级 / 完成里程碑
```

## 4. 行为驱动机制

### 4.1 宠物经验值来源

宠物经验值与现有积分系统联动，但有独立的计算公式：

| 用户行为 | 积分 (现有) | 宠物经验 | 宠物反应 |
|---------|------------|---------|---------|
| 打开App | +5 pts | +3 exp | 醒来打招呼动画 |
| 打开文件 | +2 pts | +2 exp | 探头看屏幕 |
| 阅读完成 | +10 pts | +8 exp | 鼓掌/点赞动画 |
| 创建笔记 | +8 pts | +6 exp | 开心跳跃 |
| 添加wiki-link | +5 pts | +4 exp | 连线动画 |
| NLP分析 | +3 pts | +5 exp | 思考姿态 |
| BPE分析 | +3 pts | +5 exp | 分析动画 |
| 搜索 | +1 pts | +1 exp | 放大镜动画 |
| 连续3天 | +20 pts | +15 exp | 🎉 小庆祝 |
| 连续7天 | +50 pts | +40 exp | 🎆 大庆祝 |

### 4.2 喂食系统（知识喂养）

宠物通过"知识食物"维持饱腹度。食物由学习行为产出：

| 食物 | 获取方式 | 饱腹度恢复 | 额外效果 |
|------|---------|-----------|---------|
| 📝 墨水糖 | 写 500+ 字笔记 | +20 | 快乐+5 |
| 📖 书页饼 | 完整阅读 3 篇笔记 | +15 | 经验+3 |
| 💡 灵感果 | 创建 5+ wiki-links | +25 | 亲密度+10 |
| 🔬 分析茶 | 完成 3 次NLP/BPE分析 | +10 | 好奇性格额外+5 |
| ⭐ 星辰露 | 连续7天使用 | +50 | 所有属性+10 |

### 4.3 互动系统

用户可以在App中与宠物进行简单互动：

| 互动方式 | 操作 | 效果 | 冷却 |
|---------|------|------|------|
| 🤚 摸头 | 点击宠物头部 | 快乐+5, 亲密+2 | 10分钟 |
| 🎮 玩耍 | 双击宠物 | 快乐+10, 活力-5 | 30分钟 |
| 💬 对话 | 长按宠物弹出气泡 | 随机知识小贴士 | 5分钟 |
| 🎁 送礼 | 拖拽食物到宠物 | 取决于食物类型 | 无冷却 |

## 5. 宠物技能系统

### 5.1 被动技能

宠物在成长过程中自动学习的辅助技能：

| 技能名 | 解锁条件 | 效果 |
|--------|---------|------|
| 📊 笔记提醒 | 宠物 Lv.5 | 长时间未写笔记时宠物会提醒 |
| 🔍 智能搜索 | 宠物 Lv.10 | 搜索结果旁显示相关度星级 |
| 📈 学习报告 | 宠物 Lv.15 | 每周生成学习数据可视化 |
| 🎯 目标追踪 | 宠物 Lv.20 | 设定每日学习目标并追踪 |
| 💡 知识推荐 | 宠物 Lv.30 | 基于笔记内容推荐相关文档 |
| 🏆 成就播报 | 宠物 Lv.5 | 达成成就时宠物庆祝动画 |

### 5.2 主动技能（需用户触发）

| 技能名 | 解锁条件 | 效果 | 消耗 |
|--------|---------|------|------|
| ☕ 专注模式 | 宠物 Lv.8 | 宠物安静陪伴，减少动画干扰 | 活力-10 |
| 🎲 每日挑战 | 宠物 Lv.12 | 生成一个当日学习小挑战 | 活力-5 |
| 📸 截图助手 | 宠物 Lv.18 | 宠物出现在截图合影中 | 无 |
| 🌈 主题切换 | 宠物 Lv.25 | 解锁宠物主题配色 | 无 |

### 7.3 数据存储

使用 SQLite（与现有 rusqlite 一致），表设计：

```sql
-- 宠物主表
CREATE TABLE pets (
    id TEXT PRIMARY KEY,
    owner_id TEXT NOT NULL,
    name TEXT NOT NULL,
    species TEXT NOT NULL,           -- 'ink_sprite' | 'scroll_pup' | ...
    stage INTEGER NOT NULL DEFAULT 1,
    variant TEXT DEFAULT 'base',
    happiness INTEGER DEFAULT 80,
    energy INTEGER DEFAULT 100,
    hunger INTEGER DEFAULT 100,
    bond_level INTEGER DEFAULT 0,
    exp INTEGER DEFAULT 0,
    level INTEGER DEFAULT 1,
    personality TEXT NOT NULL,        -- JSON array of traits
    mood TEXT DEFAULT 'calm',
    skills TEXT DEFAULT '[]',         -- JSON array
    accessories TEXT DEFAULT '[]',    -- JSON array
    skin_variant TEXT DEFAULT 'default',
    born_at INTEGER NOT NULL,
    last_fed_at INTEGER,
    last_interaction_at INTEGER,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

-- 宠物事件日志
CREATE TABLE pet_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    pet_id TEXT NOT NULL,
    event_type TEXT NOT NULL,        -- 'fed' | 'interact' | 'evolve' | 'skill_learn' | ...
    event_data TEXT,                 -- JSON
    created_at INTEGER NOT NULL,
    FOREIGN KEY (pet_id) REFERENCES pets(id)
);

-- 物品/食物库存
CREATE TABLE pet_inventory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    owner_id TEXT NOT NULL,
    item_type TEXT NOT NULL,         -- 'ink_candy' | 'scroll_cake' | ...
    quantity INTEGER DEFAULT 0,
    updated_at INTEGER NOT NULL
);
```

| 详情面板 | 128x128 px | 面板内大图 |
| 选蛋/进化 | 256x256 px | 全屏对话框 |
| 精灵帧 | 64x64 px | 动画每帧大小 |

### 9.2 动画规范

| 动画 | 帧数 | 循环 | 帧率 |
|------|------|------|------|
| Idle (呼吸) | 4 帧 | 循环 | 2fps |
| Walk (走路) | 6 帧 | 循环 | 4fps |
| Sleep (睡觉) | 2 帧 | 循环 | 1fps |
| React (反应) | 4 帧 | 单次 | 6fps |
| Celebrate (庆祝) | 8 帧 | 单次 | 8fps |
| Eat (吃东西) | 4 帧 | 单次 | 4fps |

### 9.3 美术风格

- **风格**：像素风 (Pixel Art) 16x16 → 放大到 64x64
- **配色**：每只宠物有主色 + 辅色 + 点缀色
- **表情**：通过眼睛变化表达情绪（点点眼/弯弯眼/× ×眼/❤️眼）
- **渲染**：SVG 或 Canvas 2D，支持 CSS 动画
- **备选方案**：若像素风制作量大，可先用 emoji + CSS 动画作为 MVP

### 9.4 MVP 精灵方案（emoji + CSS）

在 Phase 1 中，可使用组合 emoji + CSS 动画快速原型：

```
墨灵: 🐱 + ✒️ 效果
卷卷: 🐶 + 📖 效果  
码仔: 🐹 + 💻 效果
思思: 🦉 + 💭 效果
芽芽: 🐸 + 🌱 效果
```

## 12. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 宠物动画影响编辑器性能 | 高 | 使用 CSS 动画，限制帧率，编辑时降低动画频率 |
| 精灵图制作工作量大 | 中 | MVP 用 emoji+CSS，后续迭代替换为像素精灵 |
| 宠物系统过于复杂分散注意力 | 中 | 保持宠物低干扰，提供一键隐藏 |
| 存储空间膨胀 | 低 | 精灵图使用 SVG，事件日志定期清理 |
| 与积分系统耦合度高 | 中 | 通过 Reactor 模式松耦合，宠物系统可独立运行 |
