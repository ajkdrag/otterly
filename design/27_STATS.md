## 1. 设计理念

统计仪表盘是用户了解自身学习行为的核心界面。通过丰富的 SVG 图表和数据面板，展示会话统计、知识库概览、NLP 分析结果和积分成长数据。所有统计数据存储在本地 SQLite 中，后端通过 5 个 Tauri 命令提供数据。

---

## 2. 架构设计

### 2.1 目录结构

```
src/lib/features/stats/
└── ui/
    └── stats_dashboard.svelte        # 统计仪表盘组件（1226 行）
```

**注意**：stats 模块是纯 UI 模块，没有独立的 service/store/ports 层。它通过直接调用 Tauri invoke 获取数据，并使用其他模块的 store（如积分、NLP）。

### 2.2 特点

- **单文件大组件**：包含完整的仪表盘 UI + 数据获取 + SVG 图表渲染
- **无独立 index.ts**：唯一一个没有 barrel export 的 feature 模块
- **直接 Tauri 调用**：不通过端口抽象，直接 invoke 后端命令

## 3. 仪表盘面板

| 面板 | 数据来源 | 说明 |
| ---- | -------- | ---- |
| 知识成长 | `points_get_account` | 等级图标 + 称号 + 积分 + 连续天数 + 进度条 |
| Vault 概览 | `stats_scan_vault` | 文件夹/文件数 + 总大小 + 文件类型饼图（SVG） |
| 当前会话 | 本地计时器 | 动画时长 + 文件打开数 |
| 历史统计 | `stats_get_history` | 累计会话数/时长/文件打开 + 趋势图（SVG） |
| NLP 知识库 | `nlp_get_aggregate_stats` | 总词数/唯一词/关键词/词汇丰富度 |
| 积分交易 | `points_get_transactions` | 最近积分变动列表 |

---

## 4. 后端命令（Tauri）

| 命令 | 说明 |
| ---- | ---- |
| `stats_start_session` | 开始新会话 |
| `stats_end_session` | 结束当前会话 |
| `stats_file_opened` | 记录文件打开事件 |
| `stats_file_read_complete` | 记录文件读完事件 |
| `stats_get_history` | 获取历史统计数据 |
| `stats_scan_vault` | 扫描 vault 获取概览数据 |

---

## 5. SVG 图表

仪表盘使用纯 SVG 实现图表（无第三方图表库）：

| 图表类型 | 用途 |
| -------- | ---- |
| 饼图 | 文件类型分布 |
| 折线图 | 文件打开趋势 |
| 柱状图 | 每日会话统计 |
| 进度条 | 等级进度 |
| 动画计数器 | 实时会话时长 |

## 6. 依赖关系

```
stats
```
